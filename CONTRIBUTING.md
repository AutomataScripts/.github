# 🛠️ Contributing to AUTOMATA

We are glad you're here! AUTOMATA is a lab focused on efficient systems and automation. While we value high-quality engineering, we prioritize simplicity and knowledge sharing over rigid bureaucracy.

---

## 💡 Our Engineering Style

We don't have complex rules, but we appreciate it when contributions keep these points in mind:

*   **Keep it Simple**: Aim for direct and readable code. If a solution can be modular and independent, that's a win.
*   **Efficiency**: Since we work with low-level systems and infra, we always look for a conscious use of resources (CPU/Memory).
*   **Isolation**: Think about how your code runs in isolated environments, like containers or sandboxes.
*   **Consistency**: Try to follow the existing coding style of the repository you are working on.

---

## 🚀 How to Contribute

1.  **Fork & Clone**: Fork the repository you want to improve and clone it locally.
2.  **Create a Branch**: Use simple names like `fix-connection` or `new-feature-x`.
3.  **Code**: Implement your changes. Whether it's C++, Go, Python, or Bash, focus on making it robust and clean.
4.  **Pull Request**: Open a PR with a brief description of your changes. If you have test logs or screenshots, feel free to include them!

---

## 💬 Ideas and Feedback

Do you have an architectural idea, a new automation script, or an optimization suggestion? Open an **Issue**. We love discussing new ways to build independent and high-performance systems.
